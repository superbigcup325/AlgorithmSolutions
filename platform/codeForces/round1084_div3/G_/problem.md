# G. 操作排列

每组测试时间限制：2秒  
内存限制：256兆字节

AksLoI Coding 有一个整数 $x$ 和一个包含 $n$ 个操作的列表。每个操作是一个字符串，以符号 $+$、$-$、$\times$ 或 $/$（分别表示加法、减法、乘法、实数除法）之一开头，后面紧跟着一个正整数 $y$（$1 \leq y \leq 10^9$）。例如，操作 $\times 3$ 表示将 $x$ 乘以 3。

AksLoI Coding 将随机排列这些操作，然后按排列后的顺序依次对 $x$ 应用所有操作。请帮助 AksLoI Coding 计算 $x$ 的期望*最终值对 $10^9 + 7$ 取模的结果。

正式地，令 $M = 10^9 + 7$。可以证明答案可以表示为不可约分数 $\frac{p}{q}$，其中 $p$ 和 $q$ 是整数，且 $q \neq 0 \pmod{M}$。输出等于 $p \cdot q^{-1} \pmod{M}$ 的整数。换句话说，输出一个满足 $0 \leq a < M$ 且 $a \cdot q \equiv p \pmod{M}$ 的整数 $a$。

---

*$x$ 的期望最终值是所有 $n!$ 种排列下 $x$ 最终值的平均值。

## 输入
第一行包含一个整数 $t$（$1 \leq t \leq 1000$），表示测试用例的数量。

对于每个测试用例，第一行包含两个整数 $n$ 和 $x$（$1 \leq n \leq 3000$，$1 \leq x \leq 10^9$）。

每个测试用例的第二行包含 $n$ 个字符串，每个字符串代表一个操作，格式如上所述。

所有测试用例的 $n^2$ 之和不超过 $3000^2$。

**注意：** 用 $\times$ 表示乘法，而不是 $*$。

## 输出
对于每个测试用例，输出一个整数：$x$ 的期望最终值对 $10^9 + 7$ 取模的结果。

### 示例

#### 输入
```
4
2 10
x2 -10
4 2
+6 +7 /1 -13
8 1
+1 x2 x3 +4 +5 +6 -7 -8
9 864289753
-918273645 x564738291 /365107362 x734582911 -654321789 x998244353 +172519103 /482193765 /482091376
```

#### 输出
```
5
2
166666677
601980218
```

---

## 注释
在第一个测试用例中，$x$ 可以是 $(10 \times 2) - 10 = 10$ 或 $(10 - 10) \times 2 = 0$，因此期望值为 5。

在第二个测试用例中，所有可能的排列都导致 $x = 2$。

在第三个测试用例中，$x$ 的期望值为 $\frac{55}{6}$。
