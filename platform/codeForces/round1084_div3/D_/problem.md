# D. 传送门

每组测试时间限制：2秒  
内存限制：256兆字节

你有一个长度为 $n$ 的排列 $p$。还有两个传送门位于位置 $x$ 和 $y$（$x < y$）。

位置 $i$ 处的传送门最初位于数组的第 $i$ 个元素和第 $(i + 1)$ 个元素之间。具体来说，如果 $i = 0$，则传送门位于数组第一个元素之前；如果 $i = n$，则传送门位于数组最后一个元素之后。

你可以任意多次执行以下两种操作：

1. 移除一个传送门紧左边的元素，并将其插入到另一个传送门的紧右边。
2. 移除一个传送门紧右边的元素，并将其插入到另一个传送门的紧左边。

用 $O$ 表示传送门。例如，如果 $p$ 是 $[3, O, 2, 4, O, 1]$：

- 对左侧和右侧传送门分别使用操作 1 会分别得到数组 $[O, 2, 4, O, 3, 1]$ 和 $[3, O, 4, 2, O, 1]$。
- 对左侧和右侧传送门分别使用操作 2 会分别得到数组 $[3, O, 4, 2, O, 1]$ 和 $[3, 1, O, 2, 4, O]$。

找出使用这些操作可以得到的字典序 $^\dagger$ 最小的排列。请注意，传送门不影响排列的字典序比较。

---

$^\dagger$ 长度为 $n$ 的排列是一个长度为 $n$ 的数组，其中包含从 1 到 $n$ 的每个整数恰好一次。  
$^\dagger$ 如果存在一个索引 $i$，使得对于所有 $1 \leq j < i$ 都有 $a_j = b_j$，且 $a_i < b_i$，则排列 $a$ 在字典序上小于排列 $b$。

## 输入

第一行包含一个整数 $t$（$1 \leq t \leq 2 \cdot 10^4$）——测试用例的数量。

对于每个测试用例，第一行包含三个整数 $n$、$x$ 和 $y$（$1 \leq n \leq 2 \cdot 10^5$，$0 \leq x < y \leq n$）。

每个测试用例的第二行包含 $n$ 个整数 $p_1, p_2, \ldots, p_n$ ——一个长度为 $n$ 的排列。

所有测试用例的 $n$ 之和不超过 $2 \cdot 10^5$。

## 输出

对于每个测试用例，输出一行包含 $n$ 个整数 —— 你可以得到的字典序最小的排列。

## 示例

### 输入
```
4
4 0 4
3 1 4 2
3 1 2
3 2 1
5 1 3
1 3 5 2 4
2 0 1
1 2
```
### 输出

```
1 4 2 3
2 3 1
1 2 3 5 4
1 2
```

---

## 注释

用 $O$ 表示传送门。

在第一个测试用例中，数组是 $[O, 3, 1, 4, 2, O]$。对左侧传送门使用操作 2 得到 $[O, 1, 4, 2, 3, O]$，这是可以得到的字典序最小的排列。

```1 4 2 3 ```

如上所述的操作。

在第二个测试用例中，数组是 $[3, O, 2, O, 1]$。对左侧传送门使用操作 1 得到 $[O, 2, O, 3, 1]$，这是可以得到的字典序最小的排列。

在第四个测试用例中，最优选择是不执行任何操作。
