
# G. Mixing MEXes

**每个测试的时间限制：2 秒**  
**每个测试的内存限制：256 兆字节**

给定 $n$ 个数组 $a_1, a_2, \dots, a_n$。

执行以下操作**恰好一次**：

- 从 $a_1, a_2, \dots, a_n$ 中选择任意一个数组。假设你选择了数组 $a_i$（其中 $1 \leq i \leq n$）。
- 在数组 $a_i$ 中选择任意一个元素。假设你选择了 $a_i$ 的第 $j$ 个元素，记为 $a_{i,j}$（其中 $1 \leq j \leq |a_i|$，$|a_i|$ 表示数组 $a_i$ 的长度）。
- 从 $a_1, a_2, \dots, a_n$ 中选择任意一个**不是** $a_i$ 的数组。假设你选择了数组 $a_k$（其中 $1 \leq k \leq n$，且 $k \neq i$）。
- 将 $a_{i,j}$ 添加到数组 $a_k$ 的末尾，然后从 $a_i$ 中删除 $a_{i,j}$。
- 该操作 $(i, j, k)$ 的值定义为：操作后所有数组的 **MEX** 之和，即：
  $$\text{值} = \sum_{p=1}^n \text{MEX}(a_p)$$

计算**所有可能的不同操作的值之和**。两个操作被认为是不同的，当且仅当有序三元组 $(i, j, k)$ 不同。

---

## 输入格式

第一行包含一个整数 $t$（$1 \leq t \leq 10^4$），表示测试用例的数量。

每个测试用例的第一行包含一个整数 $n$（$2 \leq n \leq 2 \times 10^5$），表示数组的数量。

接下来 $n$ 行，每行先给出一个整数 $l_i$（$1 \leq l_i \leq 10^5$），表示数组 $a_i$ 的长度，接着给出 $l_i$ 个整数 $a_{i,1}, a_{i,2}, \dots, a_{i,l_i}$（$0 \leq a_{i,j} \leq 10^6$），表示数组中的元素。

保证所有测试用例的 $l_i$ 之和不超过 $2 \times 10^5$。

---

## 输出格式

对于每个测试用例，输出一个整数，表示所有可能的不同操作的值之和。

## 注意

对于第一个测试用例，我们有 3 种可能的不同操作：

- $i = 1, j = 1, k = 2$：数组变为 $[\,]$ 和 $[0, 1, 2]$，它们的 **mex** 分别是 0 和 3，所以操作的值是 3。
- $i = 2, j = 1, k = 1$：数组变为 $[0, 1]$ 和 $[2]$，它们的 **mex** 分别是 2 和 0，所以操作的值是 2。
- $i = 2, j = 2, k = 1$：数组变为 $[0, 2]$ 和 $[1]$，它们的 **mex** 分别是 1 和 0，所以操作的值是 1。

对于第二个测试用例，由于没有数组包含零，所有操作的值都将为 0。
