# C. 西蒙斯与发布博客
每组测试时间限制：2秒  
内存限制：256兆字节

---

这个文件就像一档电视节目；我们随着情节的发展被卷入其中。  
— 舜，陷阱

有 $n$ 篇博客。第 $i$ 篇博客按顺序提到了 $l_i$ 个用户，记录为一个数组 $a_i = [a_{i1}, a_{i2}, \dots, a_{il_i}]$。

你将发布所有 $n$ 篇博客。让我们维护一个序列 $Q$，它描述了你最近提到过的用户列表。你需要**恰好**执行以下操作 $m$ 次：

- 选择一个**未发布的**博客 $i$（$1 \leq i \leq n$），然后发布它。这将导致按顺序对每个 $1 \leq j \leq l_i$ 执行以下操作：
    - 如果 $a_{ij}$ 已经存在于 $Q$ 中，则将 $a_{ij}$ 移动到 $Q$ 的开头。
    - 否则，将 $a_{ij}$ 插入到 $Q$ 的开头。

找出在所有 $m$ 次操作后字典序最小的 $Q$。

*条目 $x$ 在字典序上小于或等于 $y$ 当且仅当满足以下条件之一：*

- $x$ 是 $y$ 的前缀。
- $x$ 是 $y$ 的后缀。
- 在 $x$ 和 $y$ 第一个不同的位置，$x$ 的条目比 $y$ 中对应的条目小。

---

## 输入
每个测试包含多个测试用例。第一行包含测试用例的数量 $t$（$1 \leq t \leq 1000$）。随后是测试用例的描述。

每个测试用例的第一行包含一个整数 $n$（$1 \leq n \leq 3000$）——博客的数量。

然后是 $n$ 行，第 $i$ 行以一个整数 $l_i$（$1 \leq l_i \leq 3000$）开头，描述第 $i$ 篇博客提到的用户数量，后面跟着 $l_i$ 个整数 $a_{i1}, a_{i2}, \dots, a_{il_i}$（$1 \leq a_{ij} \leq 10^6$）——第 $i$ 篇博客提到的用户列表。

保证所有测试用例的 $n$ 之和不超过 $3000$。

记 $\sum_{i=1}^n l_i$ 为 $L$。保证所有测试用例的 $L$ 之和不超过 $3000$。

---

## 输出
记 $m$ 为至少在一条博客中被提到的用户数量。对于每个测试用例，输出 $m$ 个整数 $Q_1, Q_2, \dots, Q_m$ —— 字典序最小的 $Q$。

## 注释
在第一个测试用例中，你可以按如下方式发布博客：

- 发布第一篇博客，$Q$ 将变为 $[6, 4, 3, 2, 1]$。
- 发布第三篇博客，$Q$ 将变为 $[3, 2, 9, 1, 6, 4]$。
- 发布第二篇博客，$Q$ 将变为 $[1, 5, 2, 3, 9, 6, 4]$。

还有另一种发布博客的方式：

- 发布第三篇博客，$Q$ 将变为 $[3, 2, 9, 1]$。
- 发布第一篇博客，$Q$ 将变为 $[6, 4, 3, 2, 1, 9]$。
- 发布第二篇博客，$Q$ 将变为 $[1, 5, 2, 6, 4, 3, 9]$。

我们可以看到 $[1, 5, 2, 3, 9, 6, 4]$ 在字典序上小于另一种结果。如果我们最后不发布第二篇博客，数组的第一个元素不会是 1，因此 $[1, 5, 2, 3, 9, 6, 4]$ 是字典序最小的数组 $Q$。

在第二个测试用例中，你可以按如下方式发布博客：

- 发布第一篇博客，$Q$ 将变为 $[6, 1]$。
- 发布第二篇博客，$Q$ 保持为 $[6, 1]$。

在第三个测试用例中，你只能发布唯一的一篇博客，$Q$ 将变为 $[1, 6]$。
